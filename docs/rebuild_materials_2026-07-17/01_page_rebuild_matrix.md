# Матрица пересборки страниц

## Общая структура Saby-страницы

Каждая детальная страница направления должна идти по схеме:

1. Hero: кому и зачем страница.
2. Быстрое объяснение: что это направление делает простым языком.
3. Узнаваемые ситуации: 3-4 боли клиента.
4. Что получает бизнес: результат после запуска.
5. Что входит в запуск: конкретные работы Элдокс.
6. Что влияет на цену: честный блок про расчет.
7. Частые ошибки выбора: почему нельзя просто купить "что-нибудь".
8. Как работает Элдокс: 3-5 шагов.
9. FAQ.
10. Заявка.

## Главная

| Поле | Решение |
|---|---|
| Роль | Короткая Saby-first витрина и вход в каталог направлений |
| Аудитория | Бухгалтер, руководитель малого/среднего бизнеса, сотрудник, которому поручили подключить Saby |
| Главный вопрос пользователя | "Какой Saby нам нужен и сколько это примерно будет стоить?" |
| Главный смысл | Элдокс помогает понять, какой Saby нужен под конкретную ситуацию, и запустить его без путаницы в лицензиях |
| Обязательные блоки | Hero, вход в направления, почему цена зависит от задачи, калькулятор-ориентир, короткий блок про сайты/боты как второй контур, база знаний, FAQ, заявка |
| Визуальный ритм | Hero с сильным технологичным визуалом; затем компактные продуктовые карточки; затем контрастный блок расчета; далее спокойный блок второго контура |
| CTA | "Подобрать Saby", "Рассчитать ориентир", "Найти свой случай" |
| Приемка | Пользователь за 20 секунд понимает, что сайт про Saby, видит путь в каталог и понимает, что цену уточнят по региону/задаче |

## Saby ЭДО

| Поле | Решение |
|---|---|
| Аудитория | Бухгалтер, руководитель, сотрудник документооборота |
| Ситуации | Контрагенты просят ЭДО; УПД/акты/счета теряются; статусы документов непонятны; оригиналы ищут вручную |
| Главный смысл | ЭДО нужен, чтобы документы с контрагентами были видны по статусу и не терялись между почтой, папками и людьми |
| Блоки | Hero; что такое ЭДО; ситуации; что меняется после запуска; что входит; что влияет на цену; ошибки выбора; FAQ; заявка |
| Визуал | Схема "контрагент → документ → подпись → статус → архив"; сравнение "до/после"; карточки документов |
| CTA | "Рассчитать ЭДО", "Понять состав запуска" |
| Приемка | Пользователь понимает, какие документы переводятся в ЭДО и что Элдокс делает кроме продажи лицензии |

## Saby отчетность

| Поле | Решение |
|---|---|
| Аудитория | Бухгалтер, ИП/ООО, руководитель, отвечающий за сроки |
| Ситуации | Отчеты держатся на календаре одного сотрудника; требования приходят в разные места; непонятно, что принято; несколько организаций |
| Главный смысл | Отчетность должна держать сроки, требования и сверки в понятном рабочем месте |
| Блоки | Hero; что входит в отчетность; ситуации; календарь/требования/сверки; расчет цены; запуск; FAQ; заявка |
| Визуал | Таймлайн отчетного периода; карточки "сдать", "ответить", "проверить", "получить статус" |
| CTA | "Подобрать отчетность", "Рассчитать подключение" |
| Приемка | Бухгалтер понимает, что страница про ежедневную сдачу и контроль отчетности, а не абстрактный сервис |

## Saby бухгалтерия и учет

| Поле | Решение |
|---|---|
| Аудитория | Бухгалтерия, руководитель, операционный сотрудник |
| Ситуации | Документы и счета в разных местах; ручные сверки; неясно, какие модули нужны; есть риск купить лишнее |
| Главный смысл | Бухгалтерии нужен не "комбайн", а понятный набор возможностей под ежедневную работу |
| Блоки | Hero; отличие от ЭДО/отчетности; ситуации; что можно вести; что входит в подбор; цена; FAQ; заявка |
| Визуал | Модульная карта "документы / счета / сверки / роли / поддержка"; блок "не покупаем лишнее" |
| CTA | "Подобрать состав для бухгалтерии" |
| Приемка | Пользователь понимает, зачем отдельная страница учета и чем она отличается от ЭДО и отчетности |

## Saby кадры и зарплата

| Поле | Решение |
|---|---|
| Аудитория | Кадровик, HR, руководитель, компания с растущим штатом |
| Ситуации | Заявления и приказы на бумаге; сотрудники спрашивают статусы; подписи собираются вручную; нужен КЭДО |
| Главный смысл | Кадровые документы должны проходить быстрее и понятнее для сотрудника, кадровика и руководителя |
| Блоки | Hero; что такое КЭДО/кадровые задачи; сценарий сотрудника; что входит; что влияет на цену; FAQ; заявка |
| Визуал | Сценарная линия "сотрудник → руководитель → кадры → архив"; карточки ролей |
| CTA | "Подобрать кадровое направление" |
| Приемка | Кадровик видит свои задачи, а не общий текст про цифровизацию |

## Saby торговля и склад

| Поле | Решение |
|---|---|
| Аудитория | Владелец торговой точки, управляющий, бухгалтерия торговли |
| Ситуации | Кассы, склад и документы живут отдельно; остатки сверяют вручную; УПД задерживаются; сотрудники на точках работают по-разному |
| Главный смысл | Торговля должна связывать продажу, склад, документы и учет без ручной сборки из разных мест |
| Блоки | Hero; схема торговой точки; ситуации; что входит; оборудование/кассы/склад; цена; FAQ; заявка |
| Визуал | Схема "точка продаж → склад → УПД → бухгалтерия"; карточки участков |
| CTA | "Посчитать торговлю и склад" |
| Приемка | Пользователь понимает, что страница про реальные точки продаж, а не общую автоматизацию |

## Saby маркировка

| Поле | Решение |
|---|---|
| Аудитория | Торговля, склад, производитель/продавец маркированных товаров |
| Ситуации | Нужно работать с Честным знаком/ЕГАИС/Меркурием; непонятно, какое оборудование и документы нужны; маркировка связана со складом |
| Главный смысл | Маркировка должна совпасть с товаром, складом, продажами и документами, иначе она превращается в отдельную головную боль |
| Блоки | Hero; для каких товаров/систем; ситуации; что входит; расчет; риски; FAQ; заявка |
| Визуал | Карта систем "Честный знак / ЕГАИС / Меркурий / документы / склад"; блок "что уточняем перед расчетом" |
| CTA | "Разобраться с маркировкой" |
| Приемка | Пользователь понимает, что расчет зависит от товара и схемы работы, а не от одной кнопки "подключить" |

## Сайты и CRM

| Поле | Решение |
|---|---|
| Роль | Второй контур после Saby: если вместе с Saby нужно привести в порядок заявки и продажи |
| Главный смысл | Сайт/CRM/бот не конкурируют с Saby, а закрывают соседние участки работы бизнеса |
| Требование | Не перетягивать фокус главной на агентскую витрину; подавать как расширение сотрудничества |
