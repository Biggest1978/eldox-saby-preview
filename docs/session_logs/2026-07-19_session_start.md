## Стартовая задача

Владелец уточнил фокус: сейчас не составляем реестр текущего сайта и не маппим страницы Элдокса на Tilda. Это будет следующая задача.

Текущая задача: подготовить верхнеуровневую фабричную базу знаний по блокам Tilda, чтобы следующие сборщики, арт-директор, UX и внутренний контроль могли выбирать native-блоки осознанно, а не вслепую.

## Требуемый артефакт

Документ верхнего уровня:

- объясняет, какие семейства блоков Tilda существуют и зачем они нужны;
- помогает выбрать блок под функцию страницы;
- фиксирует риски и ограничения;
- поддерживает правило фабрики: native first, custom only by explicit approval;
- отделяет текущую задачу от следующей задачи по переносу сайта Элдокс в Tilda.

После уточнения владельца задача разделена на два слоя:

1. фабричная база знаний по Tilda-блокам уже создана и сохранена как верхнеуровневый справочник;
